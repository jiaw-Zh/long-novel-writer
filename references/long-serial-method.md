# 百万/千万字长篇机制

## 长篇不是章节数放大

百万字以上的核心问题不是“写得更长”，而是让故事在长期推进中持续产生目标、代价、误解、升级和回收。默认把作品设计成多层循环：

- 整书主线：最终问题、终局真相、主角最终代价。
- 卷主线：每卷解决一个阶段性问题，同时暴露更深一层规则。
- 篇章单元：10-30 章形成一个中型目标。
- Chunk：3-5 章形成一个小悬念单元。
- 单章：一个明确推进点，一个情绪钩子，一个读者继续读的理由。

## 必备记忆层

长期连载必须维护这些文件或等价内容：

- `metadata.md`：题名、类型、基调、目标读者、篇幅目标、禁用项。
- `story-bible.md`：核心种子、主题、终局、世界观规则、叙事视角。
- `volumes.md`：分卷目标、冲突升级、卷结局、卷间钩子。
- `characters.md`：角色状态、关系网、能力/物品、秘密、心理变化。
- `worldbook.md`：世界规则、组织、地点、历史、力量体系、禁忌。
- `foreshadowing-ledger.md`：伏笔、首次出现、强化节点、回收计划、状态。
- `summaries.md`：全局摘要、卷摘要、chunk 摘要、最近章节摘要。
- `continuity-issues.md`：设定冲突、待修复问题、禁止重复桥段。

## 节奏规则

- 每 3-5 章完成一个小闭环：提出问题、推进误解、制造代价、给出阶段结果。
- 每 10-30 章完成一个篇章目标：胜利不能只是胜利，必须带出新账、新敌人或新规则。
- 每卷结尾必须回收一部分伏笔，同时打开更深层问题。
- 爽点不能只靠升级。交替使用：身份落差、信息差、规则利用、反杀、关系变化、资源夺取、真相揭露、代价承担。

## 一致性规则

写正文前检查：

- 主角此刻知道什么，不知道什么。
- 角色是否按已有动机行动。
- 本章是否违反世界规则。
- 道具、伤势、能力、地理位置是否延续。
- 当前冲突是否推进了卷目标。
- 是否重复最近 20 章用过的桥段。

写正文后更新：

- 新事实进入 `summaries.md` 或 `worldbook.md`。
- 角色变化进入 `characters.md`。
- 伏笔变化进入 `foreshadowing-ledger.md`。
- 矛盾和不确定项进入 `continuity-issues.md`。

## 检索/RAG 规则

当作品进入百万字规模，默认需要检索层。可用本地文件检索、BM25、向量库、外部 RAG API 或用户提供的知识库。用户只说“不做本地向量库/RAG”时，不要删除长期记忆需求；应保留外部 RAG API 或纯文本检索入口。

检索前生成 3-5 组关键词：

- 实体 + 属性
- 事件 + 后果
- 地点 + 特征
- 角色 + 秘密
- 伏笔 + 回收节点

检索后分类为：情节燃料、人物维度、世界碎片、叙事技法、冲突预警。
